Session Type Bank

Each session type you create has a name, category, estimated calories, and intensity. These are the building blocks — you schedule instances of them onto days of the week. Same emoji can have very different intensity (Yin vs Power Yoga).

Day Load Formula

Each session type has an intensity value from 0.0–1.0. Day load sums all sessions for the day, caps at 1.0, and maps to four labels on the green→red ramp.
The formula
// Each SessionType has intensity: Float (0.0–1.0)
// User picks from 4 presets when creating a session type:
// Low = 0.15, Moderate = 0.4, High = 0.55, Peak = 0.8

let dayLoad = min(1.0, sessions.reduce(0) { $0 + $1.sessionType.intensity })

switch dayLoad {
  case 0.0..<0.3:  "Light"    // green, 25% bar fill
  case 0.3..<0.55: "Moderate" // yellow, 50% bar fill
  case 0.55..<0.75: "High"    // orange, 75% bar fill
  default:          "Peak"    // red, 95% bar fill
}

// Calories sum directly — no cap
let totalCal = sessions.reduce(0) { $0 + $1.effectiveCalories }

// effectiveCalories = actualCalories ?? estimatedCalories
// HealthKit overwrites estimated after session completion
Intensity presets
User picks one of four when creating a session type
Low (0.15) — gentle recovery: Yin Yoga, easy walk, light core

Moderate (0.4) — solid effort: 5k run, weights session, leisure swim

High (0.55) — hard work: Power Yoga, ocean swim, hill intervals

Peak (0.8) — max output: 10k race, long run, competition swim

These are the only values the user chooses from — no free-form slider. Simple, opinionated, hard to get wrong.
Day load examples
How it adds up across the week
Mon: Yin (0.15) + Manly 5k (0.4) = 0.55 → High
Tue: Upper body (0.4) = 0.4 → Moderate
Wed: Power Yoga (0.5) + Ocean swim (0.55) = 1.0 → Peak
Thu: Harbour 10k (0.8) = 0.8 → Peak
Fri: Yin (0.15) + Core (0.15) = 0.3 → Moderate
Sat: Run w/ bro (0.4) + Fairy Bower (0.4) = 0.8 → Peak
Sun: Rest = 0.0 → Light

The AI reads these patterns: "Three peak days Wed–Sat is a lot — Friday's recovery day is well placed."

How It All Connects

Strip cards
Emoji + title only
Strip cards show day + side-by-side emoji + session names. No metadata, no calories, no intensity. The strip is for scanning the week shape at a glance. All detail lives on the hero.
Hero card
Compact: emoji pair + combined title + all metadata
Emoji pair at 38px, combined title ("Yin Yoga & Manly Loop"), subtitle with time slots + total duration + total calories. Below the divider: time-of-day pills, category type pills, day load bar. Same layout for 1 or 2 sessions — just drops to one emoji and one title.
Setting calories & intensity
Done once per session type
When you create "Manly Loop 5k", you set estimated calories (~280) and pick an intensity preset (Moderate). These are set-and-forget — they flow into every scheduled instance. HealthKit actuals overwrite the estimate when a session completes.
AI briefing reads it all
Cross-day intelligence
The AI has access to every day's load, calories, and category mix. It spots patterns: "Three peak days in a row Wed–Sat — Friday's recovery day is well placed" or "All cardio this week — Tuesday has room for a strength session." Combined with shopping metadata, it can connect energy expenditure to meal planning.
