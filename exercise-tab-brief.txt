EXERCISE TAB — IMPLEMENTATION BRIEF
=====================================

I need to add a new Exercise tab for weekly exercise planning. This follows the same patterns already established in two existing tabs: the Calendar tab (which has a Weekly Briefing feature with an AI-generated editorial summary) and the Shopping tab (which has a Dinner This Week planner at the top with horizontally scrollable day cards).

I have an HTML prototype to share as visual reference. Please follow it closely for layout, spacing, and interaction patterns.


APP CONTEXT
-----------
- SwiftUI app with dark purple/magenta theme (#1a0e2e background, #241540 card backgrounds)
- Teal (#4ecdc4) as the primary accent colour
- Uses standard emoji throughout instead of custom icons
- Existing tabs: Calendar, Shopping, Notes, Profile — Exercise is being added as a 5th tab (💪 icon)
- Claude API (Haiku model) is already integrated for the Calendar tab's weekly briefing

INTERACTION: ADDING A SESSION (bottom sheet)

STEP 1 — Pick time slot + category:
- Time slot picker: 3 horizontal pills (Morning ☀️ / Lunch 🌤️ / Arvo 🌅)
  - Default selected: Morning
  - Selected state: teal border, teal-soft background, teal text

- "Rest day" option at bottom (reduced opacity, 😴 icon)

STEP 2 — Pick session type:
- "‹ Back" link in teal

- Subtitle: "For [Day] [time]"

- Tapping a type assigns it and closes the sheet


DATA MODEL
----------

ExerciseCategory:
  - id: UUID
  - name: String (e.g. "Yoga")
  - emoji: String (e.g. "🧘")
  - colour: Color (the accent colour for this category)
  - sessionTypes: [SessionType]

SessionType:
  - id: UUID
  - name: String (e.g. "Yin Yoga")
  - description: String (e.g. "Deep stretches · 45 min · Recovery")
  - categoryId: UUID

ExerciseSlot:
  - id: UUID
  - dayOfWeek: Int (0=Monday, 6=Sunday)
  - timeSlot: TimeSlot enum (morning, lunch, arvo)
  - sessionTypeId: UUID (references a SessionType)

WeekPlan:
  - weekStartDate: Date
  - slots: [ExerciseSlot]
  - restDays: Set<Int> (day indices marked as rest)

The user creates categories and session types once, then reuses them each week by assigning them to day+time slots.


CLAUDE API INTEGRATION (exercise briefing)
------------------------------------------

This follows the EXACT same pattern as the Calendar tab's weekly briefing. Reference that implementation.

When it runs:
- Once per week on week-start day (Monday)
- Regenerates if exercise schedule changes (debounce 30 seconds)
- Shimmer animation on the briefing card while loading

What gets sent (JSON payload):
- This week's exercise slots: day, time, category name, session type name, description
- Rest days
- Category distribution (e.g. "3× yoga, 3× run, 2× swim, 1× weights, 1× pilates")
- Total session count

What comes back (JSON):
- headline: String (under 40 chars, editorial tone, uses emphasis on one phrase)
- summary: String (3-4 sentences, conversational, references actual sessions by name)

Prompt guidance for the system prompt:
- Write in a warm, editorial voice — like a magazine wellness column, not a personal trainer
- Reference the user's actual session names and days
- Comment on balance and variety — flag if one category is overrepresented or missing
- Note sequencing (e.g. recovery sessions before/after intense days)
- Suggest adjustments naturally within the prose, not as bullet points
- Keep it to 3-4 sentences maximum
- NO generic motivational filler ("You've got this!", "Great job!", "Stay strong!")
- NO coach persona or branding

Caching:
- Cache response locally, serve from cache if schedule hasn't changed
- Roughly 1-3 API calls per week per user
- Use Haiku model for speed and cost
